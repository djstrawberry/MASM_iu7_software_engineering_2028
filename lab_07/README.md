# Лабораторная работа №7: Взаимодействие C++ и Ассемблера (x86/x64)

Данный проект поддерживает сборку под обе основные архитектуры: x86 и x64.

## Функционал

1. **`my_strlen`**: Вычисление длины строки.
   - **x86**: Использование встроенного ассемблера `__asm`.
   - **x64**: Внешняя процедура в `.asm` файле.
2. **`my_copy`**: Копирование блоков памяти с защитой от затирания при перекрытии.
   - Реализована логика выбора направления копирования в зависимости от взаимного расположения адресов `src` и `dst`.

## Файловая структура

* `main.cpp` — диспетчер функций, переключающий реализации в зависимости от выбранной платформы сборки.
* `copy_x86.asm` — реализация копирования для x86.
* `copy_x64.asm` — реализация копирования для x64.
* `my_strlen_x64.asm` — реализация вычисления длины для x64.

## Настройка окружения (Visual Studio)

Для успешной компиляции вам необходимо:

1. **Build Customizations**: Правой кнопкой по проекту -> Настройки сборки -> Отметить **masm**.
2. **Платформа**: Выбрать `x86` или `x64` на верхней панели.
3. **Исключение файлов**: 
   - При сборке **x64** необходимо нажать правой кнопкой на `copy_x86.asm` -> Свойства -> Поставить **Исключить из сборки**.
   - При сборке **x86** аналогично исключить `copy_x64.asm` и `my_strlen_x64.asm`.
4. **Тип элемента**: У всех `.asm` файлов в свойствах должен стоять тип **Microsoft Macro Assembler**.

## Особенности реализации

В 64-битной версии используется соглашение о вызовах Windows x64:
* Первый аргумент (`dst`) -> `RCX`
* Второй аргумент (`src`) -> `RDX`
* Третий аргумент (`len`) -> `R8`

Логика копирования учитывает перекрытие областей памяти, переключая флаг направления (`std`/`cld`) для обеспечения целостности данных.
